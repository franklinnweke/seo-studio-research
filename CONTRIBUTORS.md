# Contributors and provenance

SEO Studio began as a PROG8751 capstone team project and was later extended into a publication-oriented research system. The public research repository preserves the original commit authors, timestamps, and application history.

The contribution areas below are based on the project charter and repository history. They summarize major areas of responsibility rather than attempting to list every commit.

## Team contributions

### Franklin Nweke

- **Project management:** sprint planning, task and issue creation, progress tracking, implementation-status records, submission coordination, and research-workstream planning.
- **Frontend development:** context-aware metadata and review interfaces, image upload and resize workflow integration, dashboard components, frontend API contracts, and interaction-state refinement.
- **Documentation and research communication:** project plans, sprint evidence, technical architecture, operational handoffs, research protocol, evaluation reports, documentation indexing, and the working manuscript.
- **Research engineering:** context and image-purpose contracts, structured metadata pipelines, evaluation-harness design, licensed dataset preparation, experiment accounting, blinding, annotation workflows, calibration analysis, and reproducibility evidence.
- **Research infrastructure and MLOps:** DGX/Ollama access coordination, model inventory and compatibility studies, runtime and truncation diagnostics, inference telemetry, resumable experiment execution, model-screening controls, and preservation of immutable experimental evidence.
- **Quality assurance and integration:** frontend, schema, OpenAPI, dataset, evidence, and protocol-consistency validation, along with database planning and shared application integration.

### Ayobami Aladenoye

- **Product and frontend development:** initial SEO Studio POC, dashboard and metadata workspaces, brand-context workflow, responsive behavior, AI metadata generation and selection, image-resizer and crop flows, metadata exports, persisted review workflow, and Supabase authentication.
- **Full-stack feature integration:** supporting API, shared-contract, sample-data, and Docker configuration changes required by frontend product flows.
- **Quality assurance:** application-flow stabilization, product usability refinement, and capstone review support.

### Michael Abelola

- **Backend and configuration support:** application configuration, health checks, environment integration, and database responsibilities established by the team charter.
- **Platform engineering and DevOps:** multi-stage Docker builds, Docker Compose orchestration, Nginx configuration, staging-server integration, GitHub Actions deployment automation, service restart policy, and deployment environment configuration.
- **Infrastructure collaboration:** helped establish the application deployment foundation later extended by the research-specific DGX/Ollama and MLOps workflow.

## Research edition

Franklin Nweke leads and maintains the research extension. Michael Abelola's earlier platform and deployment work and Ayobami Aladenoye's application and product workflows form material parts of the shared software foundation on which the research system is built.

The capstone team approved republication of the shared application foundation in `franklinnweke/seo-studio-research`. Detailed technical attribution remains available in the Git history.
